# Local Backend Development

## Requirements
- Python 3.12
- Docker Desktop
- GNU Make (or run commands manually)

## Windows Notes
- Use Docker Desktop with WSL2 backend enabled.
- Run backend tooling in the project virtual environment.

## First Run
```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install -e .[dev]
make dev
```

## Migrations
```powershell
.\.venv\Scripts\Activate.ps1
alembic upgrade head
alembic downgrade -1
alembic revision --autogenerate -m "describe change"
```

Migration rules:
- Always review autogenerated migrations before applying.
- Keep schema changes additive-first when possible.
- Never modify an already-merged migration file.
